This is a human verification checkpoint. The executor presents the verification steps below to the user and waits for approval.

**What was built:** Complete clawd-linker CLI tool with init, new, and manage commands. The tool creates a package repository, scaffolds packages, and manages per-file symlinks with state tracking, conflict detection, and idempotent re-runs.

**How to verify:**

**Setup (in a temporary location, not in the clawd-linker project itself):**

1. Create a test directory:
   ```bash
   mkdir -p /tmp/clawd-test-project && cd /tmp/clawd-test-project
   ```

2. If ~/.clawd-linker exists from a previous run, delete it:
   ```bash
   rm -f ~/.clawd-linker
   ```

**Test 1: Init (INIT-01, CFG-01)**
3. Run init from the clawd-linker project directory:
   ```bash
   cd /path/to/clawd-linker && node bin/clawd-linker.js init
   ```
4. When prompted, enter `/tmp/clawd-test-repo`
5. Verify:
   - Green success message appears
   - `/tmp/clawd-test-repo/.git` directory exists
   - `~/.clawd-linker` file exists and contains `repoPath` field
   - `cat ~/.clawd-linker` shows `{ "schemaVersion": 1, "repoPath": "/tmp/clawd-test-repo" }`

**Test 2: Init re-run warning (INIT-02)**
6. Run init again:
   ```bash
   node bin/clawd-linker.js init
   ```
7. Verify: Yellow warning message appears, no prompts shown, exits cleanly

**Test 3: New package (PKG-01, PKG-02)**
8. Scaffold a test package:
   ```bash
   node bin/clawd-linker.js new test-pkg
   ```
9. Verify:
   - `/tmp/clawd-test-repo/test-pkg/files/` directory exists
   - `/tmp/clawd-test-repo/test-pkg/PACKAGE.md` exists with `# test-pkg`
   - `/tmp/clawd-test-repo/test-pkg/data.json` exists with empty `installedIn`
   - `/tmp/clawd-test-repo/test-pkg/.gitignore` exists and contains `data.json`

**Test 4: Add files to package, then manage (MGR-01, MGR-02, MGR-03, LINK-01, LINK-02, LINK-03)**
10. Add test files:
    ```bash
    echo "test content" > /tmp/clawd-test-repo/test-pkg/files/test-file.txt
    mkdir -p /tmp/clawd-test-repo/test-pkg/files/.claude/commands
    echo "# test command" > /tmp/clawd-test-repo/test-pkg/files/.claude/commands/test.md
    ```
11. Run manage from the test project:
    ```bash
    cd /tmp/clawd-test-project && node /path/to/clawd-linker/bin/clawd-linker.js manage
    ```
12. Verify: Checkbox list shows `test-pkg` (unchecked)
13. Select `test-pkg` (space), confirm (enter), confirm proceed
14. Verify:
    - `/tmp/clawd-test-project/test-file.txt` is a symlink
    - `/tmp/clawd-test-project/.claude/commands/test.md` is a symlink
    - `ls -la /tmp/clawd-test-project/test-file.txt` shows it points to the package files/ path (absolute path)
    - Parent directory `.claude/commands/` was created (LINK-03)

**Test 5: Idempotent re-run (STATE-03)**
15. Run manage again:
    ```bash
    node /path/to/clawd-linker/bin/clawd-linker.js manage
    ```
16. Verify: `test-pkg` is pre-checked (MGR-02). Confirm without changes.
17. Verify: "No changes needed." message appears

**Test 6: Uninstall (LINK-04, STATE-01, STATE-02)**
18. Run manage again, deselect `test-pkg`, confirm
19. Verify:
    - Symlinks at `/tmp/clawd-test-project/test-file.txt` and `.claude/commands/test.md` are removed
    - `cat /tmp/clawd-test-repo/test-pkg/data.json` shows empty `installedIn`

**Test 7: Config error (CFG-02)**
20. Delete the test repo:
    ```bash
    rm -rf /tmp/clawd-test-repo
    ```
21. Run manage:
    ```bash
    node /path/to/clawd-linker/bin/clawd-linker.js manage
    ```
22. Verify: Red error message about repo not found, with actionable instructions

**Cleanup:**
```bash
rm -f ~/.clawd-linker
rm -rf /tmp/clawd-test-repo /tmp/clawd-test-project
```
